Architecture

Patterns and decisions behind the projects.

Batch Pipeline — Olist dbt Pipeline

The Olist project follows a layered architecture where each stage has a single responsibility. Raw data lands untouched in staging, transformations happen progressively through intermediate models, and marts deliver clean, business-ready tables.

Scraper + API — PriceTrail

PriceTrail separates concerns across three services: the scraper collects data, PostgreSQL stores it, and FastAPI exposes it. Docker Compose ties everything together, making the environment reproducible with a single command.

Dimensional Modeling — Finance DW

The Finance DW project implements a star schema with a central fact table and four surrounding dimension tables. Grain is defined at the individual transaction level.

Principles

  • Layered Architecture — each stage has a single responsibility
  • Reproducibility — Docker and documented dependencies
  • Code Versioning — all projects on GitHub with CI where applicable
  • Decision Documentation — scope, assumptions, and choices recorded in READMEs
